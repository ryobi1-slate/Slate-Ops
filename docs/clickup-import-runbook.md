# ClickUp Import Runbook

## 1. Purpose

One-time procedure to migrate active ClickUp Schedule jobs into the Slate Ops
database after deployment. The importer tooling lives in PR #128 (merged);
this runbook documents the on-server workflow that uses it.

The procedure is **destructive** to the `slate_ops_jobs` table — it clears
existing jobs and replaces them with rows derived from the ClickUp export.
Users, roles, settings, and `work_centers` are never touched.

## 2. Required files

| File | Source | Notes |
|---|---|---|
| `local-imports/clickup-active-cleaned.jsonl` | Generated by `local-imports/build-jsonl.py` from the ClickUp CSV export | Gitignored — uploaded to the server out-of-band |
| `local-imports/run-import.php` | In the repo (merged via PR #128) | CLI runner; bootstraps WordPress and dispatches each step |

Supporting files (also in the repo, no action needed):

- `includes/cli/class-clickup-importer.php` — backup / clear / import logic
- `backup/.gitkeep` — placeholder so `backup/` exists for the backup step

## 3. Where files must live on the WordPress server

Let `<plugin-root>` be the directory containing `slate-ops.php` — typically
`/var/www/html/wp-content/plugins/slate-ops/`.

```
<plugin-root>/
├── local-imports/
│   ├── clickup-active-cleaned.jsonl   ← upload here (SFTP/SCP/rsync)
│   └── run-import.php                 ← already on server (deployed)
├── includes/cli/
│   └── class-clickup-importer.php     ← already on server (deployed)
└── backup/
    └── jobs-pre-clickup-import.json   ← written by the backup step
```

Upload the JSONL with e.g.:

```bash
scp clickup-active-cleaned.jsonl \
    user@server:/var/www/html/wp-content/plugins/slate-ops/local-imports/
```

Verify:

```bash
wc -l /var/www/html/wp-content/plugins/slate-ops/local-imports/clickup-active-cleaned.jsonl
# expect: 23
```

All commands below are run from `<plugin-root>` on the WordPress server.
Substitute the actual `--wp-path` if WordPress lives somewhere other than
`/var/www/html`.

## 4. Backup command

Snapshots every existing job row to `backup/jobs-pre-clickup-import.json`.
Non-destructive.

```bash
php local-imports/run-import.php \
    --wp-path=/var/www/html \
    --step=backup
```

Confirm `backup/jobs-pre-clickup-import.json` exists and is non-empty before
continuing.

## 5. Clear dry-run command

Counts what `clear-execute` would delete. Makes no changes.

```bash
php local-imports/run-import.php \
    --wp-path=/var/www/html \
    --step=clear-dry
```

Sanity-check the reported job / time-segment / audit counts against what
the live CS page shows.

## 6. Import dry-run command

Parses the JSONL and shows what would be inserted. Makes no changes.

```bash
php local-imports/run-import.php \
    --wp-path=/var/www/html \
    --step=import-dry
```

Output must match section 9 exactly before proceeding.

## 7. Clear execute command

Deletes all jobs and related time segments / audit rows.

```bash
php local-imports/run-import.php \
    --wp-path=/var/www/html \
    --step=clear-execute --confirm
```

`--confirm` is required; the runner refuses to write without it.

## 8. Import execute command

Inserts the rows from `clickup-active-cleaned.jsonl`.

```bash
php local-imports/run-import.php \
    --wp-path=/var/www/html \
    --step=import-execute --confirm
```

`--confirm` is required.

## 9. Expected import counts

Before running `import-execute`, `import-dry` must report:

| Metric | Expected |
|---|---|
| Active rows | **23** |
| Rows with `so_number` populated | **23** |
| Rows missing `so_number` | **0** |

If any number differs, **stop and investigate** — do not run execute.

After `import-execute` the runner output should read:

```
Inserted: 23
Skipped:  0
```

## 10. Verification checklist

After `import-execute` completes, verify on the live CS page:

- [ ] CS page shows ~23 active jobs.
- [ ] Closed/historical jobs from ClickUp are hidden (not present in the list).
- [ ] `so_number` is visible on every job row.
- [ ] Statuses are mapped correctly:
  - Scheduled — 7
  - In Progress — 4
  - Delayed — 9
  - Complete - Awaiting Pickup — 3
- [ ] Parts statuses: Ready — 10, Not Ready — 13.
- [ ] Users, roles, settings, and `work_centers` are unchanged from before
      the clear step.

## 11. Rollback

A pre-clear snapshot is written by step 4 (backup) at:

```
<plugin-root>/backup/jobs-pre-clickup-import.json
```

This is a JSON array of every row that was in `slate_ops_jobs` at backup
time. Restore is **manual and DB-aware** — there is no automated rollback
script. The recommended path is:

1. Take a fresh full DB dump (`wp db export` or `mysqldump`) before any
   restore attempt, so the post-import state is preserved.
2. With a DBA (or someone comfortable with direct SQL), reconstruct the
   `INSERT` statements from `jobs-pre-clickup-import.json` — the JSON keys
   map 1:1 to the `slate_ops_jobs` columns.
3. Truncate `slate_ops_jobs` and replay the inserts inside a transaction.
4. Re-verify the CS page against the pre-import job list.

Do not run a restore without a current DB dump in hand.
